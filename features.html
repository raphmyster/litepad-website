Features & Guide

Everything Litepad does is built around one idea: markdown files should open like readable documents first, with editing controls available when you need them.

Preview Mode

The default experience. Open any .md file and Litepad renders it with proper headings, spacing, code blocks, and lists.

Litepad opens in Preview by default. Just double-click any markdown file in Finder and start reading.
preview.md

Split View

See the markdown source and the rendered preview side by side. It is useful when reviewing AI-generated files or checking structure without losing readability.

Click Split in the toolbar to keep source on one side and preview on the other.

Edit Mode

When you need to make changes, Litepad switches into a focused source editor. The formatting toolbar helps with headings, emphasis, lists, tables, and more.

Click Edit to switch to source mode, then use the toolbar for quick formatting while you write.

Dark Mode & Themes

Litepad follows your system appearance and supports theme choices for people who prefer lower glare, especially during long reading sessions.

Open theme settings in the app preferences to follow system mode or choose a theme manually.

Set Litepad as Your Default Markdown App

If you want every markdown file to open in Litepad automatically, macOS lets you switch the default app for the .md file type in a few clicks.

  1. Right-click any .md file in Finder.
  2. Choose Get Info.
  3. Under Open With, select Litepad.
  4. Click Change All to apply it to every markdown file.

Print & Export to PDF

Markdown does not have to stay inside markdown workflows. Litepad can turn a clean preview into a printable document or exported PDF when you need to share it.

Use File → Print or press ⌘P to print or save the rendered document as a PDF.
